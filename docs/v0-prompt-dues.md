# v0 프롬프트 — 회비 관리 페이지 리디자인

## 프롬프트

다음은 풋살/축구 팀 관리 앱(PitchMaster)의 **회비 관리 페이지** 디자인 요청입니다.
다크 모드 전용이며, 모바일 퍼스트(375px~430px)로 디자인해주세요.

### 브랜드 컬러
- Primary(코랄): hsl(16, 85%, 58%) — #e8613a
- Background: hsl(240, 6%, 6%)
- Card: hsl(240, 5%, 10%)
- Success(납부완료): hsl(152, 55%, 55%)
- Loss(미납): hsl(0, 65%, 60%)
- Warning(면제/주의): hsl(38, 85%, 58%)
- Info(정보): hsl(210, 70%, 60%)
- Muted text: hsl(40, 5%, 62%)

### 전체 구조

회비 페이지는 **4개 탭**으로 구성됩니다 (underline 스타일 탭 바, flex-1 균등 분배):
1. **입출금** — 월별 입출금 내역 (전원 조회 가능)
2. **납부현황** — 회원별 납부 상태 (운영진+)
3. **내역 올리기** — OCR/엑셀 업로드 (운영진+)
4. **설정** — 회비 기준 설정 (운영진+)

평회원은 "입출금" 탭만 보이고, 상단에 본인 납부 상태 카드가 표시됩니다.

---

### Section 1: 상단 잔고 카드 (탭 바 위, 항상 표시)

```
┌─────────────────────────────────┐
│ 회비 현황                        │
│                                 │
│ ┌─ featured card ─────────────┐ │
│ │ 통장 잔고                    │ │
│ │ 1,193,000원        (큰 글씨) │ │
│ │ 최종 업데이트: 3월 28일 09:00 │ │
│ │                              │ │
│ │ ── 월별 수지결산 ──           │ │
│ │ 4월  수입 +360,000           │ │
│ │      지출 -150,000           │ │
│ │      순이익 +210,000원        │ │
│ └──────────────────────────────┘ │
└─────────────────────────────────┘
```

- 잔고 숫자는 `type-score` 크기 (clamp 2rem~3rem), primary 색상
- "통장 잔고"는 `type-overline` (작은 라벨)
- 월별 수지결산은 border-top으로 구분, 수입(success색), 지출(loss색), 순이익(bold)
- 잔고가 없으면 "내역 올리기 탭에서 스크린샷이나 엑셀을 올리면 잔고가 반영됩니다" 안내

### Section 1-1: 평회원 전용 — 내 납부 상태 카드

```
┌─────────────────────────────────┐
│ 2026.04 내 납부 상태    < 04 >  │
│                                 │
│ 납부 완료              30,000원 │
│ (초록색 텍스트)                  │
│                                 │
│ 또는                            │
│ 미납                   [입금했어요] │
│ (빨간색 텍스트)          (버튼)  │
└─────────────────────────────────┘
```

- 월 좌우 이동: `< >` 화살표 (min-h 44px 터치 타겟)
- 납부 완료: success 색상, 미납: loss 색상, 면제: warning 색상
- "입금했어요" 버튼은 미납일 때만 표시 (outline variant)

### Tab 1: 입출금 탭

```
┌─────────────────────────────────┐
│ 입출금 내역              < 04 > │
│                                 │
│ [전체 ▼]  [수기 입력 폼 토글]    │
│                                 │
│ ── 수기 입력 (접기/펼치기) ──    │
│ [유형: 입금▼] [금액] [내용]     │
│ [회원 선택▼] [날짜]  [저장]     │
│                                 │
│ ── 거래 내역 ──                 │
│ ┌──────────────────────────────┐│
│ │ 2026-04 회비 (김데모)        ││
│ │ 2026-04-05 · 김데모          ││
│ │                    +30,000원 ││
│ │              (입금: 초록 뱃지)││
│ └──────────────────────────────┘│
│ ┌──────────────────────────────┐│
│ │ 2026-04 구장 대여비          ││
│ │ 2026-04-10                   ││
│ │                   -150,000원 ││
│ │              (지출: 빨간 뱃지)││
│ └──────────────────────────────┘│
│ ...                             │
└─────────────────────────────────┘
```

- 월별 네비게이션: `< 2026.04 >` (화살표 + 월 표시)
- 필터: "전체/입금/지출" Select
- 수기 입력 폼: 접기/펼치기 토글 (운영진+만 표시)
- 거래 내역 리스트: 카드 스타일
  - 설명 (font-semibold)
  - 날짜 · **멤버 이름** (이름만 bold + foreground/80)
  - 금액: 입금 → success 뱃지 (+금액), 지출 → loss 뱃지 (-금액)
  - 카드 간 간격: space-y-2.5
- 빈 상태: EmptyState 컴포넌트 ("아직 거래 내역이 없습니다")

```
│ ── 장기 미납자 (3개월+) ──      │

│                                 │
│ ── 회원별 현황 ──               │
│ ┌──────────────────────────────┐│
│ │ 김데모    일반    납부 완료   ││
│ │                    30,000원  ││
│ │         [납부▼]              ││
│ └──────────────────────────────┘│
│ ┌──────────────────────────────┐│

│ └──────────────────────────────┘│
│ ...                             │
└─────────────────────────────────┘
```

- 상단 월별 네비, 납부 기간 표시
- "납부 자동 확인" / "미납 알림" 버튼
- 납부 통계: 프로그레스 바 (success 색상) + 텍스트 요약
- 장기 미납자 경고: loss 배경 카드
- 회원별 리스트: 이름, 회비 유형, 납부 상태 (Select로 변경 가능)

### 디자인 가이드라인

1. **카드**: rounded-xl (16px), bg-card, border border-white/4, subtle shadow
2. **Featured 카드**: 코랄 gradient overlay (135deg, 15% opacity)
3. **텍스트 최소 크기**: 12px (text-xs)
4. **터치 타겟**: 최소 44px
5. **탭 바**: sticky top-0, underline 스타일 (border-b-2 border-primary), flex-1 균등 분배, backdrop-blur
6. **금액 색상**: 입금/납부완료 → success, 지출/미납 → loss
7. **빈 상태**: EmptyState 컴포넌트 (아이콘 + 제목 + 설명 + CTA 버튼)
8. **스켈레톤**: 로딩 중 shimmer 효과
9. **간격**: 섹션 간 gap-5, 카드 내부 space-y-2.5
10. **폰트**: Pretendard Variable (본문), Bebas Neue (스코어/숫자)
