## 1. 앱 개요

**앱명**: PitchMaster
**카테고리**: 스포츠 / 팀 관리
**대상 사용자**: 풋살·축구 동호회 운영진 및 회원
**핵심 가치**: 경기 투표·출석·득점·회비·전술판을 한곳에서 관리

## 3. 테스터 피드백으로 발견·수정한 주요 이슈

테스터가 실제 사용 중 보고한 이슈를 기반으로 아래 수정을 완료했습니다.

### 3-1. 크리티컬 버그 수정

| # | 이슈 | 원인 | 수정 |
|---|------|------|------|
| 1 | 다른 팀 경기에 투표가 들어감 | 투표 API에서 경기의 team_id 검증 누락 | `match.team_id !== 사용자 teamId`일 때 403 반환 추가 |
| 2 | 수동 마감한 투표 재개 시 마감일이 +1개월로 설정 | 임시 디폴트값이 배포됨 | 경기 전날 17시 KST로 복원하는 로직으로 교체 |
| 3 | 자동 편성 시 휴면 회원이 포함됨 | status 필터에 DORMANT 제외 없음 | 자동 편성 쿼리에서 휴면 멤버 제외 |
| 4 | 감독 지정 포지션을 운영진이 저장할 수 없음 | PRESIDENT 전용 권한으로 잘못 분류 | MATCH_EDIT 권한(STAFF 이상)으로 분리 |
| 5 | 자동 완료 경기가 당일 종료 경기를 놓침 | match_end_time 조건 누락 | 당일 종료 시간이 지난 경기도 포함하도록 수정 |
| 6 | 사전등록 멤버가 휴면 전환되어도 투표 대상에서 제외 안 됨 | 투표 카운트 필터 누락 | 활성 멤버 기준으로 정규화 |

### 3-2. 보안 강화

| # | 항목 | 내용 |
|---|------|------|
| 1 | 세션 쿠키 Secure 플래그 | 프로덕션 환경에서 HTTPS 전용 쿠키로 강제 |
| 2 | 세션 HMAC 서명 | SESSION_SECRET 기반 HMAC-SHA256 서명으로 쿠키 변조 방지 |
| 3 | API 팀 소속 검증 | 투표·출석·경기 수정 API에 크로스 팀 접근 차단 |
| 4 | 초대코드 가입 일원화 | 권한 우회 가능성이 있던 MANUAL 팀 경로를 team_join_requests로 통합 |

### 3-3. UX 개선 (테스터 피드백 반영)

| # | 피드백 | 대응 |
|---|--------|------|
| 1 | "프로필 사진이 없어서 누구인지 헷갈린다" | 카카오 프로필 자동 연동 + 커스텀 업로드 + 삭제 기능 / 헤더·회원 목록·게시판 3곳 노출 |
| 2 | "PC에서 경기 공유가 안 된다" | Windows 공유 모달 대신 클립보드 복사 + prompt 폴백 |
| 3 | "전술판에서 이 선수가 이 포지션에 적합한지 모르겠다" | 슬롯 포지션과 일치하는 선수 하이라이트 + 감독 지정 포지션 우선 적용 |
| 4 | "선수 카드가 밋밋하다" | 홀로그래픽 애니메이션, sparkle, rarity별 디자인 차별화로 카드 시각 개선 |
| 5 | "커리어 프로필이 허전하다" | OVR 산정 + 시즌 어워드 + 베스트 모먼트 섹션 추가 |
| 6 | "용병 정보를 경기 생성 후 수정할 수 없다" | 전술 탭에서 용병 편집 기능 추가 |
| 7 | "골 기록 순서가 시간순이 아니라 헷갈린다" | 등록 순(created_at) 정렬로 변경 |

### 3-4. 성능 개선

| # | 항목 | 효과 |
|---|------|------|
| 1 | 경기 상세 탭 3개 Lazy Loading | 초기 번들 축소, TBT(Total Blocking Time) 감소 |
| 2 | 날씨 API 서버 prefetch | LCP render delay 제거 |
| 3 | Hydration mismatch 제거 | React Error #418 해결 |
| 4 | browserslist 최신화 | Legacy JS 14KB 감소 |
| 5 | 폰트 최적화 | Geist 제거 + Bebas Neue 비차단 로드 |
| 6 | 댓글 서버 prefetch | 초기 상호작용 지연 감소 |
