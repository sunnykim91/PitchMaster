# v0 추가 프롬프트 2차 — 회비 페이지 마무리 보완

아래 7가지 항목을 수정/추가해주세요. 기존 레이아웃은 유지합니다.

---

## 1. 납부현황 — "미납" 텍스트 중복 제거

현재 미납 회원 카드에 "미납 20,000원" 텍스트 + "미납" 뱃지가 동시에 표시되어 "미납"이 2번 나옵니다.

수정: 금액 앞 "미납" 텍스트를 제거하고, 아래 구조로 통일해주세요:

```
┌──────────────────────────────────┐
│ 이테스트  학생         미납 (뱃지) │
│                       20,000원   │
│              [미납 ▼]            │
└──────────────────────────────────┘
```

- 우측 상단: 상태 뱃지만 표시 (납부 완료: success, 미납: loss, 면제: warning)
- 우측 하단: 금액만 표시 (미납이면 loss 색상, 납부 완료면 muted-foreground)
- Select는 뱃지 아래에 배치

---

## 2. 프로그레스 바 — 그라데이션 → 단색

납부 통계 프로그레스 바가 아직 초록→파랑 그라데이션입니다.

수정:
- 바 fill: 단색 `hsl(152, 55%, 55%)` (success)
- 바 배경: `hsl(0, 0%, 20%)` 또는 muted/30
- 바 높이: h-3, rounded-full
- 그라데이션 제거

---

## 3. 설정 탭 — 회비 금액 콤마 포맷

현재 "30000 원", "20000 원"으로 콤마 없이 표시됩니다.

수정:
- 표시: `30,000원`, `20,000원` (콤마 + "원" 붙임, 공백 없음)
- 입력 시: 숫자 입력하면 자동으로 콤마가 들어가는 형태
- 금액 텍스트: `font-semibold text-foreground`

---

## 4. 수기 입력 폼 — 펼친 상태 디자인

현재 "수기 입력" 토글이 접힌 상태만 보입니다. **펼친 상태를 기본으로 보여주세요:**

```
┌─ 수기 입력 (펼침) ──────────────────┐
│                                      │
│ 유형              금액               │
│ ┌──────────┐    ┌─────────────────┐ │
│ │ 입금  ▼  │    │ ₩       30,000  │ │
│ └──────────┘    └─────────────────┘ │
│                                      │
│ 내용                                 │
│ ┌──────────────────────────────────┐ │
│ │ 4월 회비                         │ │
│ └──────────────────────────────────┘ │
│                                      │
│ 회원 선택            날짜            │
│ ┌──────────────┐  ┌──────────────┐  │
│ │ 김데모    ▼  │  │ 2026-04-07   │  │
│ └──────────────┘  └──────────────┘  │
│                                      │
│ ┌──────────────────────────────────┐ │
│ │              저장                 │ │
│ └──────────────────────────────────┘ │
└──────────────────────────────────────┘
```

- 모든 Input/Select: `h-12 rounded-xl bg-secondary border-0`
- 유형 "입금" 선택 시 Select 텍스트 success 색상, "지출" 선택 시 loss 색상
- 유형이 "지출"이면 "회원 선택" 필드 숨기기
- 금액 Input: 좌측에 "₩" 접두사, 우측 정렬(text-right), 숫자 입력 시 콤마 자동 포맷
- 저장 버튼: w-full, primary 색상, h-12

## 5. 내역 올리기 — OCR 인식 결과 테이블

현재 업로드 영역만 있고, 업로드 후 결과 화면이 없습니다. 사진 업로드 후 표시될 결과 테이블을 **업로드 영역 아래에 예시로 함께 보여주세요:**

```
┌─ 인식 결과 (3건) ───────── [전체 저장] ┐
│                                         │
│ ┌─ 1번째 ─────────────────────────────┐│
│ │ 날짜        유형          금액       ││
│ │ [04-05]    [입금 ▼]    [₩ 30,000]  ││
│ │ 내용                    회원 매칭    ││
│ │ [4월 회비]             [김데모 ▼]   ││
│ │                          [삭제 🗑️]  ││
│ └─────────────────────────────────────┘│
│                                         │
│ ┌─ 2번째 ⚠️ 중복 의심 ──────────────┐ │
│ │ 기존 내역과 중복될 수 있습니다      │ │
│ │ (04-05, 30,000원, 김데모)          │ │
│ │                                     │ │
│ │ 날짜        유형          금액      │ │
│ │ [04-05]    [입금 ▼]    [₩ 30,000] │ │
│ │ 내용                    회원 매칭   │ │
│ │ [4월 회비]             [김데모 ▼]  │ │
│ │                                     │ │
│ │   [건너뛰기]    [그래도 저장]       │ │
│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

- 일반 행: bg-card, rounded-xl, border border-white/4
- 중복 의심 행: `border-[hsl(38,85%,58%)]/30 bg-[hsl(38,85%,58%)]/5` (warning 테두리+배경)
- 중복 안내 텍스트: text-xs, warning 색상
- "건너뛰기": ghost 버튼, "그래도 저장": outline 버튼
- 모든 Input/Select: h-10 rounded-lg bg-secondary border-0
- "전체 저장" 버튼: primary, 우측 상단

## 6. 평회원 뷰 — 내 납부 상태 카드

운영진 뷰와 별도로, **평회원이 볼 때의 화면**을 추가 섹션으로 디자인해주세요:

- 잔고 카드는 동일하게 표시
- 탭 바: "입출금" 탭 하나만 표시
- 잔고 카드와 입출금 탭 사이에 "내 납부 상태" 카드 추가:

```
┌─────────────────────────────────────┐
│ 내 납부 상태             < 04 >     │
│                                     │
│ ── 납부 완료 상태 ──                │
│ ✅ 납부 완료                30,000원│
│ (success 색상, font-semibold)       │
│                                     │
│ ── 미납 상태 ──                     │
│ ❌ 미납                    30,000원 │
│ (loss 색상)       [입금했어요 💬]   │
│                   (outline 버튼)    │
│                                     │
│ ── 면제 상태 ──                     │
│ 면제 (부상 휴식)                    │
│ (warning 색상, italic)              │
└─────────────────────────────────────┘
```

- 3가지 상태(납부완료/미납/면제) 중 해당하는 것만 표시
- "입금했어요" 버튼: outline variant, 미납일 때만 노출
- 월 이동 화살표: min-h-[44px] min-w-[44px] 터치 타겟

## 7. 빈 상태 + 스켈레톤 로딩

### 빈 상태 (입출금 탭 예시)

입출금 탭에 데이터가 하나도 없을 때:

```
┌─────────────────────────────────────┐
│          📋 (큰 아이콘)              │
│                                     │
│     아직 거래 내역이 없습니다        │
│  (text-muted-foreground, text-sm)   │
│                                     │
│  내역 올리기 탭에서 스크린샷이나     │
│  엑셀을 올려 거래 내역을 추가하세요  │
│                                     │
│       [내역 올리기] (primary)        │
└─────────────────────────────────────┘
```

- 아이콘: 48px, text-muted-foreground/50
- 제목: text-base font-semibold
- 설명: text-sm text-muted-foreground, text-center
- CTA 버튼: primary variant

### 스켈레톤 (입출금 탭 예시)

로딩 중일 때:

```
┌─ 잔고 카드 스켈레톤 ───────────────┐
│ ░░░░░░░░░░ (h-4 w-16)              │
│ ░░░░░░░░░░░░░░░░░░░ (h-10 w-48)   │
│ ░░░░░░░░░░░░░░ (h-3 w-32)          │
│ ─────────────────                   │
│ ░░░░░░░░░░░ (h-4 w-full) × 3줄    │
└─────────────────────────────────────┘

┌─ 거래 카드 스켈레톤 × 3개 ─────────┐
│ ░░░░░░░░░░░░ (h-4 w-40)            │
│ ░░░░░░░░ (h-3 w-24)     ░░ (h-6)  │
└─────────────────────────────────────┘
```

- Skeleton 컴포넌트: rounded-lg, shimmer 애니메이션
- 카드 3개 반복으로 리스트 느낌
