## 맨 위 소개 (수정)

6년간 디자인 시스템 툴링, 키오스크, CMS 등 다양한 프로덕트를 처음부터 설계하고 운영해왔습니다.

"이건 왜 이렇게 불편하지?"에서 출발해 직접 만들어보는 편입니다. 4년간 풋살 팀을 운영하면서 매주 반복되던 비효율을 없애고 싶어 시작한 사이드 프로젝트가 현재 42개 팀 · 180명 이상이 쓰는 서비스가 됐습니다. 기획부터 개발, 운영, 사용자 피드백 반영까지 혼자 돌리면서 "만드는 것"과 "쓰이는 것" 사이의 차이를 줄이는 데 많이 배웠습니다.

복잡하거나 반복되는 문제를 단순한 구조로 정리하는 걸 좋아하고, 팀 안에서도 그런 역할을 자연스럽게 해온 것 같습니다.

## PitchMaster 섹션 (수정)

### PitchMaster — 조기축구·풋살 팀 관리 플랫폼
2026.02 - 현재 | 1인 기획·개발·운영

실제 서비스 : https://www.pitch-master.app

풋살 팀 총무를 4년 하면서, 매주 금요일마다 카톡으로 참석 여부 묻고, 통장 캡쳐를 엑셀에 옮기고, 경기장 도착해서 라인업을 짜는 게 반복됐습니다. "이 세 가지만 자동화되면 총무 업무의 80%가 사라지겠다"는 생각에서 시작했고, 현재 42개 팀 · 180+ 회원이 실제로 사용하고 있습니다.

**은행마다 다른 회비 데이터를 하나의 흐름으로 정리**
회비 정산이 가장 까다로웠습니다. 총무들이 올리는 입력이 통장 스크린샷, 엑셀 파일, 수기 입력 세 가지인데, 은행마다 포맷이 다르고 OCR 결과도 들쑥날쑥합니다. 처음엔 케이스별로 나눠서 처리했는데 금방 한계가 왔고, 결국 "입력이 뭐든 → 날짜·금액·이름 목록으로 통일 → 팀원 이름과 자동 매칭"이라는 하나의 흐름으로 묶었습니다. 한 자리 날짜(3.27), 콤마가 포함된 금액, 중복 거래 같은 예외는 통일하는 단계에서 걸러지니까, 새로운 은행 포맷이 들어와도 파서 하나만 추가하면 됩니다. 월 30분 수기 작업이 30초로 줄었습니다.

**9개 페이지에 반복되던 데이터 처리를 훅 하나로 통일**
서비스가 커지면서 모든 페이지에 "서버에서 초기 데이터를 가져오고 → 클라이언트에서 실시간으로 갱신하는" 같은 패턴이 반복됐습니다. 페이지마다 조금씩 다르게 구현되어 있다 보니 서버/클라이언트 데이터가 안 맞는 버그가 여기저기서 터졌고요. `useApi(url, initialData)` 훅 하나로 서버 초기값 전달, 60초 캐싱, 데이터 변경 시 자동 갱신, 앱 복귀 시 백그라운드 갱신을 한 곳에 모았습니다. 9개 페이지의 데이터 로딩 코드가 거의 같은 모양으로 정리되면서 데이터 불일치 버그도 사라졌습니다.

**라인업 자동 배치 — 조건이 많은 배치 문제를 3단계로 단순화**
선수마다 선호 포지션이 다르고, 쿼터별로 출전 시간을 균등하게 나눠야 하고, 감독이 특정 선수의 포지션을 고정할 수도 있습니다. 처음엔 복잡하게 느껴졌는데, 결국 "고정 배치 먼저 채우고 → 나머지를 선호도 점수 순으로 채우고 → 쿼터 간 출전 횟수를 맞추는" 3단계로 정리하니 20분 걸리던 라인업이 3초 만에 나옵니다. 사용자들이 가장 좋아하는 기능이기도 합니다.

**실제 운영하면서 배운 것들**
만들어놓고 끝이 아니라, 실제 사용자 환경에서 부딪히는 문제들이 많았습니다. 카카오톡 인앱브라우저에서 세션 쿠키가 안 넘어가서 투표가 안 되는 문제, 관리자 대시보드에서 팀별 통계를 하나씩 조회해서 3초씩 걸리던 문제(병렬 조회 + 인덱스 추가로 0.5초로 개선), 투표 버튼을 누르고 응답이 올 때까지 화면이 멈춘 것처럼 보이는 문제(먼저 화면에 반영하고, 실패하면 되돌리는 방식으로 해결) 같은 것들입니다. 이런 경험을 통해 "동작하는 코드"와 "사용자가 자연스럽게 쓸 수 있는 코드"는 다르다는 걸 느꼈습니다.

**기타**
- Web Push 알림 직접 구현 (Service Worker → 구독 관리 → Cron으로 투표 마감 리마인더 발송)
- 자체전(A팀 vs B팀) 모드: 팀 편성, 별도 스코어보드, 전적 반영 ON/OFF 등 복합 상태 관리
- Vitest 615개 테스트, Vercel 자동 배포, PWA(홈 화면 추가 + 오프라인)

기술 스택: Next.js 16, React 19, TypeScript, Supabase(PostgreSQL), TailwindCSS 4, Vercel, Web Push(VAPID)
