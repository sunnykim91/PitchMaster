# PitchMaster 블로그 시리즈 기획

## 시리즈 개요
- 제목: "풋살 팀 총무가 직접 만든 팀 관리 서비스" (가제)
- 플랫폼: velog (추천)
- 대상 독자: 사이드 프로젝트에 관심 있는 개발자, 1인 개발자
- 편수: 4편

---

## 1편: 왜 만들었나 — 총무 4년의 고통에서 시작된 사이드 프로젝트
- 상태: 초안 작성 완료 (blog-post-1-draft.md)
- 핵심: 공감 스토리 + 기술 선택 이유 + 첫 버전까지의 과정
- 키워드: 사이드 프로젝트, 풋살, 팀 관리, Next.js, Supabase

## 2편: 기술적으로 까다로웠던 것들
- 상태: 초안 작성 완료 (blog-post-2-draft.md)
- 다룰 내용:
  - 은행마다 다른 회비 데이터 → 하나의 처리 흐름으로 통일한 과정 (OCR + 파싱 + 매칭)
  - SSR/CSR 데이터 불일치 → useApi 훅으로 9개 페이지 패턴 통일
  - 라인업 자동 배치 알고리즘 (우선순위 큐 + 쿼터 밸런싱)
  - 카카오 인앱브라우저 세션 쿠키 삽질기
  - 코드 스니펫 포함 권장

## 3편: 0명에서 180명 — 사용자를 어떻게 모았나
- 상태: 미작성
- 다룰 내용:
  - 네이버 카페 홍보 (어떤 카페, 어떤 글 형식, 반응)
  - 첫 번째 외부 팀이 가입한 순간
  - 데모 모드 도입 전후 전환율 차이
  - 총무한테 먹히는 메시지 ("30분 → 30초")
  - 소셜프루프 효과 (팀 수 표시 후 가입률 변화)
  - 랜딩페이지 개선 히스토리

## 4편: 혼자서 서비스 운영하기
- 상태: 미작성
- 다룰 내용:
  - 1인 개발에서 CS 대응, 버그 수정, 기능 요청 밸런스
  - 실제 사용자 피드백으로 바뀐 것들 (온보딩 필수필드 5개→2개 등)
  - 615개 테스트를 쌓은 이유
  - 지금까지의 숫자들 (팀 수, 회원 수 추이 그래프)
  - 앞으로의 방향

## 참고
- 각 편 분량: 2000~3000자 (읽는 시간 5~7분)
- 이미지: 실제 서비스 스크린샷, 아키텍처 다이어그램 포함 권장
- 상호 링크: 각 편 하단에 시리즈 다른 글 링크 추가
