## 2. 채널별 실행 계획

### 2-1. velog / 티스토리 (개발 블로그)
**목적**: 개발자 커뮤니티에 인지도 + 기술 백링크
**빈도**: 격주 1편 (무리하면 지속 안 됨)

- [ ] **1편 게시** — [blog-post-1-draft.md](docs/blog-post-1-draft.md) 그대로 velog 업로드
  - 태그: `사이드프로젝트` `풋살` `축구` `Next.js` `Supabase` `1인개발`
  - 본문 하단에 `pitch-master.app` 링크 1회 (자연스럽게)
- [ ] **2편 게시** — [blog-post-2-draft.md](docs/blog-post-2-draft.md) 업로드
  - 코드 스니펫 포함 (OCR 파싱, useApi 훅 등)
- [ ] **3편 작성** — "0명에서 180명" 성장 이야기 (plan 기준)
- [ ] **4편 작성** — 혼자 운영하기 (CS·테스트·버그)
- [ ] 각 편 게시 후 **Search Console 에 URL 수동 색인 요청** (블로그 URL)

> velog 추천: 개발자 유입·검색 노출·코드 친화. 티스토리는 네이버 검색 우회에 도움.

### 2-2. 네이버 조기축구 카페
**목적**: 실사용자 유입 (총무·회장)
**빈도**: 카페당 1회 (반복 글은 삭제당함)

- [ ] 대상 카페 확인 + 가입 등급 확보 (글 쓰려면 등업 필요한 경우 많음)
  - 모두의풋살
  - 조기축구연합
  - 풋살가이드
  - 지역별 조기축구 카페 (예: 서울 조기축구 연합)
- [ ] [marketing-cafe-post.md](docs/marketing-cafe-post.md) **경험담 형식**으로 올림 — 광고 티 나면 즉시 블랙
- [ ] 링크는 글 **맨 마지막 한 줄에 자연 안내** ("궁금하시면: pitch-master.app")
- [ ] 댓글 반응 체크 — 3일 내 피드백 대응
- [ ] 광고글 신고 당하면 해당 카페 재시도 금지, 다른 카페로

### 2-3. 개발자 SNS (X/Twitter · Threads)
- [ ] 블로그 편마다 게시물 스레드 — 1편은 왜 만들었는지 + 링크
- [ ] 해시태그: #사이드프로젝트 #NextJS #Supabase
- [ ] 관련 개발자 계정 몇 개 태그 (친분 있는 한에서만)

### 2-4. 오픈카톡방
- 조기축구 총무·회장 오픈카톡방 — "이런 앱 써보세요" 형태로 가볍게
- 본인 팀 운영 커뮤니티가 먼저

### 2-5. 플레이 스토어 메타 (간접 SEO)

## 3. 절대 하지 말 것 (SEO 역효과)

- ❌ 링크팜 / 유료 백링크 — 구글 알고리즘 역점수
